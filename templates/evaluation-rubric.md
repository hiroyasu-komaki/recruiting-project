# 評価基準（ルーブリック）

> このファイルはスクリーニングレポートにおける評価軸・評価基準の定義です。
> `/cv-screening` スキルはレポート生成前に必ずこのファイルを読み込み、
> 以下の定義に従って評価を行ってください。

---

## 項目スコアの定義（5段階）

| スコア | 判定 | 定義 |
|-------|------|------|
| **5** | 極めて上 | 要求を大きく上回る具体的な実績・経験・スキルがCVに明記されている |
| **4** | 要求以上 | 要求を満たし、かつそれを上回る経験・実績がCVに確認できる |
| **3** | 充足 | CVに具体的な経験・実績の記述があり、要件を満たしていると判断できる |
| **2** | 要求を満たしていない | 関連する経験はあるが、年数・規模・深度・役割が要件に対して不足している、またはCVの記述が曖昧で要件充足の確認が必要な状態 |
| **1** | 記載なし | CVに該当する経験・スキルの記述がない |

---

## 総合評価（A・B・C・D）の定義と判定基準

| 評価 | ラベル | 定義 |
|------|--------|------|
| **A** | 強く推薦 | 必須要件の全項目平均が3.5以上であり、かつスコア2以下の項目が一つもない状態 |
| **B** | 推薦 | 必須要件の全項目平均が3.0超であり、要件を概ね満たしている状態（Aの基準は満たさない） |
| **C** | 条件付き推薦 | 必須要件の全項目平均が2.5以上〜3.0以下。または、平均3.0超であってもスコア1や2の項目が含まれており、面接での補完確認が必要な状態 |
| **D** | 見送り推奨 | 必須要件の全項目平均が2.5未満であり、ポジションの要求水準に達していない状態 |

---

## 判定フロー

```
1. 各必須要件項目を1〜5のスコアで評価する

2. 全必須要件項目の「スコア平均値」を算出し、「スコア1・2の有無」を確認する

3. 以下の条件に従って総合評価を判定する

   - 平均 < 2.5 の場合 → D（見送り推奨）

   - 2.5 ≤ 平均 ≤ 3.0 の場合 → C（条件付き推薦）

   - 平均 > 3.0 の場合：

      - 必須要件の中にスコア1または2が含まれる → C（条件付き推薦） ※平均値は高いが致命的な懸念事項があるため
      - スコア1・2が一つもなく、平均が 3.5 以上である → A（強く推薦）
      - 上記のいずれにも該当しない → B（推薦）

```

## 評価軸の定義

### 必須要件のスコアリング対象
- job-descriptionの「必須要件」セクションに記載された各要件を個別に評価する
- 要件名はjob-descriptionの記述に準じて抽出する（要約・統合・省略は行わない）
- 「根拠・コメント」欄にはCVの具体的な記述（プロジェクト名・期間・役割・成果）を根拠として記載する
- スコアは必須要件のみを使用して総合評価を算出する

### 歓迎要件の扱い
- job-descriptionの「尚可（あれば望ましい）」セクションの要件は参考スコアとして記載する
- 歓迎要件のスコアは総合評価（A・B・C）の算出には使用しない
- 歓迎要件の充足状況は「強み」「面接での確認推奨事項」の記述に反映させる

### 強み
- 職種要件との対応を明示したうえで記述する
- 「〜の経験が、本職種の〜要件に対して有効に機能する」という視点で記述する
- CVに記述のない推測・仮定は含めない

### 懸念点・不足事項
- スコア1・2の要件のうち、採用判断に影響するものを記述する
- 「記載なし」のみを懸念点とする場合は、その要件の職種上の重要度を添える
- 候補者を否定的に評価するためではなく、面接での確認点を明確にする目的で記述する

### 面接での確認推奨事項
- CVから読み取れない情報の確認（英語力・資格・経験の詳細等）
- スコア2の要件に対する深掘り質問
- 候補者のモチベーション・キャリア観に関する確認（ポジションとのミスマッチ懸念がある場合）
- 1項目につき1つの具体的な質問または確認事項として記述する

## 全体サマリーの記述基準

- 「一言コメント」は30字以内で、総合評価の根拠が端的に伝わる表現にする
- 「総評」は2〜3文で、全職種を俯瞰した候補者の特徴・強み・採用上のポイントを記述する
- 「最も適性が高い職種」は総合評価が最上位の職種を選択する。同位の場合はスコア平均が高い職種を選択する
