候補者のCVを全募集職種と照合してスクリーニングレポートを生成します。

## 手順

### Step 1: inputフォルダのファイル検査

`input/` フォルダ内にあるすべてのファイルをリストアップし、拡張子でPDF／非PDFを仕分けしてください。

**非PDFファイル（.docx / .doc / .gdoc / .odt など）の処理:**
- `error/` フォルダが存在しない場合は作成する
- 非PDFファイルを `input/` から `error/` フォルダへ移動する（Bashツールで `mv` を使用）
- 移動が完了したら、以下のフォーマットでユーザーに通知する：

```
⚠️ 読み込み不可ファイルを検出しました

以下のファイルはPDF形式ではないため処理をスキップし、error/ フォルダへ移動しました：
- {ファイル名1}（{形式}）
- {ファイル名2}（{形式}）
```

**PDFファイルが1件もない場合:**
- 「input/ フォルダにPDFファイルが見つかりませんでした。処理を終了します。」と通知して終了する。

### Step 2: PDFファイルの読み込み

検査を通過したPDFファイルを1件ずつ Readツールで読み込み、内容をメモリに保持してください。各PDFは独立した候補者として扱い、データを混在させないこと。

### Step 3: 評価対象の確認とユーザー承認

`job-descriptions/` フォルダ内のすべての `*_job-description.md` ファイルを読み込み、各ファイルの「募集職種」名を抽出してください。

以下のフォーマットで処理内容を表示し、実行の確認を求めてください：

```
【スクリーニング実行内容の確認】

対象候補者: {N}名
評価対象職種（全職種を対象に評価します）:
  - {職種名1}
  - {職種名2}
  - {職種名3}

評価を実行しますか？ (yes / no)
```

- ユーザーが「no」と回答した場合は処理を終了する
- ユーザーが「yes」と回答した場合は Step 4 へ進む

### Step 4: スクリーニング評価と出力（候補者ごとに繰り返す）

**重要: 候補者1名につき必ず1ファイルを出力すること。複数候補者の評価結果を1ファイルに混在させてはならない。**

各PDFファイル（候補者）について、以下を個別に実行してください：

**4-0. テンプレートと評価基準の読み込み**

レポート生成前に以下の2ファイルをReadツールで必ず読み込み、体裁・章立て・評価基準を確認すること：

- `templates/screening-report-template.md`：レポートの構造・テーブル形式・マークダウン記法・体裁ルール
- `templates/evaluation-rubric.md`：A〜D評価の定義・充足状況の判定基準・評価軸の定義・判定フロー

レポートはこれらに完全に準拠して生成すること。job-descriptionの評価項目（必須要件・歓迎要件の具体的内容）は各募集要項から動的に抽出するが、体裁・評価軸・評価基準・文体はテンプレートと評価ルーブリックで固定する。

**4-1. 全職種に対する評価**

すべての job-description.md ファイルと照合し、職種ごとに以下を評価する：
- **必須要件の充足度**: 募集要項の必須スキル・経験との一致度
- **歓迎要件の充足度**: あれば加点となるスキル・経験との一致度
- **経験年数**: 求められる経験年数との比較
- **強み**: 職種に対して特に評価できるポイント
- **懸念点**: 不足しているスキルや経験
- **職種別評価**: A（強く推薦）/ B（推薦）/ C（条件付き推薦）/ D（見送り推奨）

**4-2. ファイル名の生成**

5桁のランダムな数字列をファイル名として生成する（例: `38271`）。
候補者の氏名・元ファイル名はファイル名に含めてはならない。
ファイル名: `{5桁ランダム数字}_screening-report.md`

**4-3. レポートの書き出し**

`output/` フォルダに1ファイルとして出力する。
レポートの構造・テーブル形式・マークダウン記法・文体は `templates/screening-report-template.md` に完全に準拠すること。
評価値の決定は `templates/evaluation-rubric.md` の判定フローに従うこと。

---

### Step 5: 完了報告

全候補者の処理が完了したら、以下のサマリーを報告してください。
**報告にファイル名は含めてよいが、ファイル名と候補者名の対応は記載しないこと。**

```
✅ スクリーニング完了

処理結果:
  - 処理済みCV: {N}件
  - スキップ（非PDF）: {N}件
  - 評価職種数: {N}職種

出力ファイル（output/）:
  - {5桁}.md
  - {5桁}.md
```
