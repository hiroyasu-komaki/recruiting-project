候補者のCVを全募集職種と照合してスクリーニングレポートを生成します。

## 厳守事項

個人情報の保護とログ出力の制限
候補者のCV（テキストデータ）には機密性の高い個人情報が含まれます。
ファイルを読み込む際や、AIが内部でPythonスクリプトを生成・実行してデータを取得する際、**標準出力（コンソール・ターミナルログ）にCVの本文やJSONの生データを絶対に出力（print）しないでください。**
ファイルの読み込み結果は変数やメモリ内にのみ保持し、ユーザーへの途中経過の報告は「〇〇のファイルを読み込みました」という状態報告のみにとどめてください。

## 手順

### Step 1: Python前処理スクリプトの実行

以下のBashコマンドを実行し、結果をJSONとして受け取ってください：

```bash
./src/venv/bin/python ./src/preprocess.py
```

このスクリプトは以下を自動処理します：
- `input/` フォルダのPDF／非PDFを仕分け
- 非PDFファイルを `error/` フォルダへ移動
- 全PDFのテキストを抽出

**JSONの構造:**
```json
{
  "candidates": [
    { "id": "38271", "filename": "xxx.pdf", "pages": 6, "text": "..." }
  ],
  "moved_to_error": [
    { "filename": "xxx.docx", "format": ".docx" }
  ],
  "extraction_errors": [
    { "filename": "yyy.pdf", "error": "..." }
  ]
}
```

**非PDFファイルが移動された場合:**
以下のフォーマットでユーザーに通知する：

```
⚠️ 読み込み不可ファイルを検出しました

以下のファイルはPDF形式ではないため処理をスキップし、error/ フォルダへ移動しました：
- {ファイル名1}（{形式}）
- {ファイル名2}（{形式}）
```

**`candidates` が空の場合:**
「input/ フォルダにPDFファイルが見つかりませんでした。処理を終了します。」と通知して終了する。

**`extraction_errors` がある場合:**
エラーが発生したPDFのファイル名とエラー内容をユーザーに通知し、そのファイルはスキップして処理を続行する。

---

### Step 2: 募集要項・テンプレートの読み込み

以下のBashコマンドを実行し、結果をJSONとして受け取ってください：

```bash
./src/venv/bin/python ./src/load_context.py
```

このスクリプトは以下を自動処理します：
- `job-descriptions/` フォルダ内の全 `*_job-description.md` ファイルの読み込み（raw text）
- `templates/` フォルダ内の全テンプレートファイルの読み込み

**`error` フィールドが含まれるエントリがある場合:**
該当ファイルの読み込み失敗をユーザーに通知し、そのファイルはスキップして処理を続行する。

---

### Step 3: 募集要項の構造化マッピング

以下のBashコマンドを実行してください：

```bash
./src/venv/bin/python ./src/map_context.py
```

このスクリプトは以下を自動処理します：
- Step 2 で生成された `json/context.json` の各 `job_descriptions` エントリの `content`（raw text）を読み込む
- `templates/job-description-template.json` のスキーマに従い、募集要項を要素分解して構造化JSONにマッピング
- テンプレート情報（`templates` 配列）も引き継ぎ、`json/context_mapping.json` として保存・出力

**JSONの構造:**
```json
{
  "job_descriptions": [
    {
      "filename": "xxx_job-description.md",
      "structured": {
        "募集要項": {
          "募集職種": { "職種名": "...", "レベル": "..." },
          "現状の課題": [{ "テーマ": "...", "詳細": ["..."] }],
          "期待する役割・貢献": [{ "テーマ": "...", "詳細": ["..."] }],
          "必須要件": { "実務経験": ["..."], "専門知識・スキル": ["..."] },
          "尚可": { "実務経験": ["..."] }
        },
        "業務レベル要件定義": {
          "業務レベルの評価軸": {},
          "レベル定義": [],
          "業務スコープ定義": []
        },
        "appendix": "..."
      }
    }
  ],
  "templates": [
    { "filename": "screening-report-template.md", "content": "..." },
    { "filename": "evaluation-rubric.md", "content": "..." }
  ]
}
```

**`error` フィールドが含まれるエントリがある場合:**
該当ファイルのマッピング失敗をユーザーに通知し、そのファイルはスキップして処理を続行する。

各 `job_descriptions` エントリの `structured.募集要項.募集職種.職種名` から募集職種名のリストを抽出する。
**※Readツールによるファイルの再読み込みは不要。このJSONを唯一のソースとして使用すること。**

---

### Step 4: 評価対象の確認とユーザー承認

以下のフォーマットで処理内容を表示し、実行の確認を求めてください：

```
【スクリーニング実行内容の確認】

対象候補者: {N}名
評価対象職種（全職種を対象に評価します）:
  - {職種名1}
  - {職種名2}
  - {職種名3}

評価を実行しますか？ (yes / no)
```

- ユーザーが「no」と回答した場合は処理を終了する
- ユーザーが「yes」と回答した場合は Step 5 へ進む

---

### Step 5: スクリーニング評価と出力（候補者ごとに繰り返す）

**重要: 候補者1名につき必ず1ファイルを出力すること。複数候補者の評価結果を1ファイルに混在させてはならない。**

Step 1 で取得した `candidates` リストの各エントリ（`text` フィールド）を候補者のCV情報として使用し、以下を個別に実行してください：

**5-1. 全職種に対する評価**

Step 3 で生成した `context_mapping.json` の `job_descriptions` を照合対象として使用する。
各エントリの `structured` フィールド（構造化JSON）を募集要項情報として参照し、職種ごとに以下を評価する：
- **必須要件の充足度**: `structured.募集要項.必須要件` の各サブセクションとの一致度
- **歓迎要件の充足度**: `structured.募集要項.尚可` との一致度
- **経験年数**: 求められる経験年数との比較
- **強み**: 職種に対して特に評価できるポイント
- **懸念点**: 不足しているスキルや経験
- **職種別評価**: A（強く推薦）/ B（推薦）/ C（条件付き推薦）/ D（見送り推奨）

評価値の決定は `context_mapping.json` 内 `templates` → `evaluation-rubric.md` の `content` に記載された判定フローに従うこと。

**5-2. ファイル名の決定**

Step 1 の JSON で当該候補者に割り当てられた `id` フィールド（5桁の数字）をそのままファイル名に使用する。
新たに乱数を生成してはならない。候補者の氏名・元ファイル名はファイル名に含めてはならない。
ファイル名: `{id}_screening-report.md`

**5-3. レポートの書き出し**

`output/` フォルダに1ファイルとして出力する。
レポートの構造・テーブル形式・マークダウン記法・文体は `context_mapping.json` 内 `templates` → `screening-report-template.md` の `content` に完全に準拠すること。

---

### Step 6: 完了報告

全候補者の処理が完了したら、以下のサマリーを報告してください。
**報告にファイル名は含めてよいが、ファイル名と候補者名の対応は記載しないこと。**

```
✅ スクリーニング完了

処理結果:
  - 処理済みCV: {N}件
  - スキップ（非PDF）: {N}件
  - 評価職種数: {N}職種

出力ファイル（output/）:
  - {5桁}.md
  - {5桁}.md
```
